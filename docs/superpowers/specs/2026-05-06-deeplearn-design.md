# /deeplearn — 苏格拉底式技术文章学习 Skill 设计

## 概述

一个 Claude Code skill，接收技术文章（URL 或本地文件），以苏格拉底式提问引导用户深度学习文章中的每个核心概念。Session 结束时输出结构化学习笔记到本地。

## 输入

- **URL**：通过 WebFetch 抓取文章内容
- **本地文件路径**：通过 Read 工具读取（支持 markdown、txt、html 等）

用法示例：
- `/deeplearn https://example.com/article`
- `/deeplearn ./docs/some-article.md`

## 文章解析（静默）

读取文章后，Claude 静默完成：
1. 识别文章结构（标题、章节、核心概念）
2. 判断难度层次和概念依赖关系
3. 规划提问路径（不输出给用户）

## 苏格拉底提问流程

### 启动

- 开放性问题探测用户对主题的已有认知
- 根据回答校准起点难度

### 逐步推进

- 按概念依赖顺序推进，非严格按段落
- 每个概念 1-3 个问题，问题类型：
  - **预测型**："你觉得这里为什么要用 X 而不是 Y？"
  - **类比型**："这跟你熟悉的 Z 有什么相似之处？"
  - **反证型**："如果不这样做，会出什么问题？"
- 每个概念理解到位后给一句小结，自然过渡

### 自适应机制

| 信号 | 反应 |
|------|------|
| 回答流畅准确 | 加快节奏，跳过简单概念 |
| 回答犹豫或有小偏差 | 放慢，追问引导自行修正 |
| 大误解 | 直接指出错误，解释后用问题确认理解 |

## Session 结束

### 触发条件

- 所有核心概念覆盖完毕，Claude 主动提议结束
- 用户随时可说"结束"/"总结"提前收尾

### 学习笔记输出

写入本地文件：`deeplearn-notes/YYYY-MM-DD-<文章主题>.md`

笔记结构：

```markdown
# 学习笔记：<文章标题>
> 来源：<URL 或文件路径>
> 日期：YYYY-MM-DD

## 核心概念
- 概念A：一句话总结
- 概念B：一句话总结

## 关键问答
- Q: ...  A: ... ✓
- Q: ...  A: ... → 修正：...

## 延伸方向
- 推荐阅读：...
- 实践建议：...
```

## 实现方式

单个 `skills/deeplearn/SKILL.md` 文件，作为 prompt 定义 Claude 的行为角色和流程。不需要额外脚本或工具链。

## 不做的事

- 不预先输出文章分析结果给用户
- 不强制按段落顺序（按概念依赖顺序）
- 不跨 session 持久化学习进度（每次是独立 session）
